# Архитектура zapret-next (текущее состояние)

## Компоненты

- `general*.bat` — профили стратегий запуска `winws.exe`.
- `service.bat` — интерактивный менеджер: установка/удаление службы, диагностика, тесты, обновления.
- `bin/` — бинарные зависимости (`winws.exe`, WinDivert, fake payloads).
- `lists/` — доменные и IP-списки для фильтрации/исключений.
- `utils/` — служебные файлы (флаги, targets, вспомогательные скрипты).
- `scripts/` — инженерные скрипты валидации и проверки целостности.

## Основные потоки

1. **Standalone-режим**
   - Пользователь запускает `general*.bat`.
   - Скрипт загружает состояние через `service.bat` (статус, user lists, game filter).
   - Запускается `bin/winws.exe` с параметрами текущей стратегии.

2. **Service-режим**
   - Через `service.bat` выбирается стратегия.
   - Аргументы из выбранного bat-файла подготавливаются для `sc create`.
   - Служба `zapret` стартует автоматически.

3. **Диагностика**
   - `service.bat` проверяет системные зависимости/конфликты (BFE, прокси, WinDivert, конфликтующие сервисы и т.д.).

## Ограничения текущей архитектуры

- Стратегии дублируются по множеству bat-файлов.
- Установка службы зависит от парсинга bat-строк.
- Диагностика преимущественно текстовая, без machine-readable отчётов.
